Stage 1

impDemographic_1234

Raw Data Storage
This is the initial landing zone where unprocessed client file data is stored. Think of it as the "inbox" for your import. The data here is exactly as received from the client with no modifications or transformations applied.

Stage 2

impDemographicTransformation_1234

Mapper & Transformations
The raw data is processed through your mapper logic in this stage. All IF/THEN decision trees, field mappings, data validations, and business rule transformations are applied here. This is where client-specific configurations come into play.

Stage 3

impDemographicActualData_1234

Current System Data (Comparison Base)
This table contains the data currently existing in the bswift system. It serves as the baseline for comparison operations to identify deltas (changes) between the new import data and what's already in production. This comparison determines what records need to be inserted, updated, or terminated.
üîç Comparison Table ‚Üí Used to Detect Changes (Deltas)
-- Query to view current system state
SELECT * FROM impDemographicActualData_1234;
-- Compare against Transformation table to identify deltas

Stage 4

impDemographicCommit_1234

Ready for Production Commit
This is the final staging area containing validated, approved data ready to be committed to the DB1 production databases. Once data reaches this table and passes all quality checks, it's ready to "go live" and update the production system.
‚úÖ Final Validated Data ‚Üí Commit to DB1 Production
-- Query to view data ready for production commit

-- This data will be pushed to DB1 production databases

üîë Universal Structure

Every import type follows this same 4-table pattern, ensuring consistency across all data import processes.

üìä Import ID

Each import receives a unique ID (e.g., 1234) that's appended to all four table names for that specific import run.

üîÑ Sequential Process

Data flows sequentially through all four stages. Each stage depends on the successful completion of the previous stage.

Quick Reference: To work with any import type, simply replace "Demographic" with your import type name (Enrollment, Credits, COBRA, etc.) and use your specific import ID. The 4-stage structure remains identical across all import types.
